Two editions, one app

Switch anytime in Settings → Appearance. Same Shields, sync, and page tools — different chrome.

Classic

Calm teal chrome, serif titles, and a start page that stays out of the way.

Pulse

Obsidian studio look, Pulse Corner, Data Saver, ambience, and Mac governors — no fake CPU meters.

Oriel Pulse

Built for play. Still private.

A darker studio edition inside the same binary — Corner controls, performance limits, local ambience, and workspace presets. Same privacy model as Classic. No Opera-style account.

  • Pulse Corner — launch, savers, ambience, mute, hibernate
  • Data / Network Saver, Lucid Mode, Low Power Mode tightening
  • Mac CPU & RAM governors (timer throttle + memory-pressure hibernate)
  • Optional Pulse home-screen / Dock icon

What you get

Everyday browsing tools that work in Classic and Pulse.

  • Tabs & library — groups, search, Reading List, bookmarks, history, download history that survives relaunch
  • Page tools — find with match counts, mute tab, screenshot, Save as PDF, zoom remembered per site, Reader, translate, print
  • Profiles & workspaces — separate cookie jars; Play / Stream / Focus presets
  • Extensions — Chrome Web Store, Firefox Add-ons, and Safari package import via Oriel Store (WebExtensions where the OS allows)
  • Password Vault — optional AES-GCM vault with Keychain-wrapped key and Touch ID unlock; system Keychain autofill still available
  • iCloud sync — bookmarks, Reading List, open tabs, limited history, appearance (not vault secrets)

Best of both engines

On Mac, the default is Smart (best per tab): stubborn web apps get Chromium Compatible (Chrome UA and Client Hints on WebKit — not Blink); Apple and captcha-sensitive hosts stay on WebKit. Lock every tab, set per-site overrides, or choose Chromium Native — embedded CEF when installed, otherwise a managed Chromium app-window (real Blink process).

iPhone and iPad stay on WebKit only. That’s Apple’s rule — stated honestly, not papered over.

Oriel Shields

Bundled content-blocker lists — EasyList, AdGuard-derived filters, DuckDuckGo tracker hosts — plus cosmetics, HTTPS upgrades, tracking-parameter strip, and Fire. Honest about WebKit limits: some first-party stacks still slip through.

See docs/PRIVACY.md for what stays local vs Keychain vs iCloud.
